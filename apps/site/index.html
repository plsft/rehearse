// CI SDK

YAML is a dead language for CI/CD.

@gitgate/ci compiles TypeScript to GitHub Actions YAML on your machine. Type-safe pipelines, IDE autocomplete, shared functions, conditional logic — none of which YAML gives you. The output is plain YAML you can read, diff, and ship without GitGate at runtime.

TYPE-SAFE
No more `cannot read property of undefined`

Steps, runners, secrets — all typed. Your editor catches typos before you push.

SHARED FUNCTIONS
Reuse across repos like real code

Define a deploy step once. Import it everywhere. Refactor without YAML copy-paste.

UBICLOUD DEFAULT
10× cheaper than GH-hosted

Defaults to ubicloud-standard-4. Switch to GitHub-hosted in one line.

conditional matrix — TypeScript 
const nodes = isPR ? ['20'] : ['18', '20', '22'];

job('test', {
  runner: Runner.ubicloud('standard-4'),
  matrix: { variables: { node: nodes }, failFast: false },
  steps: [step.checkout(), node.setup('\${{ matrix.node }}'), node.test()],
});

// MIGRATE

Already have YAML? Convert it.

Point gg ci convert at any existing GitHub Actions workflow. It parses the YAML, maps runner labels to Ubicloud, rewrites actions/checkout and other common steps to typed builders, and emits a TypeScript file you can refactor like normal code.

.github/workflows/ci.yml — before 
name: CI
on:
  pull_request:
    branches: [main]
  push:
    branches: [main]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm test
.gitgate/pipelines/ci.ts — after 
// AUTO-CONVERTED from GitHub Actions YAML by @gitgate/ci
import { Runner, job, pipeline,
         step, triggers } from '@gitgate/ci';

export const workflow = pipeline('CI', {
  triggers: [
    triggers.pullRequest({ branches: ['main'] }),
    triggers.push({ branches: ['main'] }),
  ],
  jobs: [
    job('test', {
      runner: Runner.ubicloud('standard-4'),
      steps: [
        step.checkout(),
        step.action('actions/setup-node@v4',
          { with: { 'node-version': '20' } }),
        step.run('npm ci'),
        step.run('npm test'),
      ],
    }),
  ],
});
