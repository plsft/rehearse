# node-app — Node + Vitest matrix CI

A tiny Node library with three exports and matching tests, plus a CI pipeline
that runs the test suite across **Node 18.x / 20.x / 22.x in parallel** via
`@rehearse/runner`'s per-cell `git worktree` isolation.

## What this example demonstrates

| Feature | Where |
| --- | --- |
| `strategy.matrix` parallel execution | `node-version: [18.x, 20.x, 22.x]` |
| Per-cell git worktree (no shared-workspace races) | how `runner` runs the matrix |
| `actions/cache` with `restoreKeys` | `~/.npm` cached on `package-lock.json` hash |
| `actions/upload-artifact` per matrix cell | coverage report uploaded per Node version |
| `hashFiles()` typed expression helper | `@rehearse/ci`'s context API |

## Pipeline at a glance

```typescript
import { hashFiles, job, pipeline, Runner, step, triggers } from '@rehearse/ci';
import { node } from '@rehearse/ci/presets';

export const ci = pipeline('Node app CI', {
  triggers: [triggers.pullRequest(), triggers.push({ branches: ['main'] })],
  jobs: [
    job('test', {
      runner: Runner.github('ubuntu-latest'),
      matrix: {
        variables: { 'node-version': ['18.x', '20.x', '22.x'] },
        failFast: false,
      },
      steps: [
        step.checkout(),
        node.setup('${{ matrix.node-version }}'),
        step.cache({
          path: '~/.npm',
          key: `npm-\${{ runner.os }}-${hashFiles('**/package-lock.json')}`,
          restoreKeys: ['npm-${{ runner.os }}-'],
        }),
        node.install(),
        node.test(),
        step.uploadArtifact({
          name: 'coverage-${{ matrix.node-version }}',
          path: 'coverage/',
          ifNoFilesFound: 'warn',
        }),
      ],
    }),
  ],
});
```

## Run locally

```bash
cd examples/node-app

# Install the example's runtime deps
npm install

# Install the Rehearse toolchain (one-time, global)
npm install -g @rehearse/runner @rehearse/cli

# Compile the TS pipeline → .github/workflows/ci.yml
rh ci compile

# Run the workflow locally — three Node versions in parallel
runner run .github/workflows/ci.yml
```

Expected: three matrix cells (`node=18.x`, `node=20.x`, `node=22.x`) run
concurrently. Each gets its own `git worktree` checkout off `HEAD`, with
`node_modules` symlinked through. Total wall-clock is the longest cell, not
the sum of the three.

## Push to GitHub

Push the repo to GitHub. The compiled YAML uses `runs-on: ubuntu-latest`,
so the job runs on standard GitHub-hosted runners with no additional
configuration.

To target a different runner pool — bigger vCPU tiers, self-hosted, or
a third-party hosted runner provider — swap `Runner.github('ubuntu-latest')`
for `Runner.github('ubuntu-latest-4-cores')`, `Runner.selfHosted(...)`, or
`Runner.custom('your-label')` in `ci.ts` and recompile with `rh ci compile`.
