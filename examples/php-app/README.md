# php-app — PHP + PHPUnit + PHPStan matrix CI

A small PHP library with PHPUnit tests and PHPStan static analysis. The CI
pipeline runs against **PHP 8.2, 8.3, and 8.4 in parallel** using
`shivammathur/setup-php@v2` — a JavaScript action that
`@rehearse/runner` auto-clones and executes locally without pre-installing
any PHP toolchain on the host.

## What this example demonstrates

| Feature | Where |
| --- | --- |
| Remote JavaScript action via runner's JS-action runtime | `shivammathur/setup-php@v2` (`runs.using: node20`) |
| Auto-clone of remote actions at the requested ref | runner caches into `.runner/actions/shivammathur__setup-php__v2/` |
| `strategy.matrix` parallel execution via per-cell git worktree | `php-version: [8.2, 8.3, 8.4]` |
| Step outputs across steps | `composer-cache.outputs.composer-cache-dir` flows into the cache step |
| `actions/cache` keyed on PHP version + composer.json hash | composer download cache, hits across rebuilds |
| Step-level `id:` for output capture | `step.run('...', { id: 'composer-cache' })` |

## Pipeline at a glance

```typescript
job('test', {
  runner: Runner.github('ubuntu-latest'),
  matrix: {
    variables: { 'php-version': ['8.2', '8.3', '8.4'] },
    failFast: false,
  },
  steps: [
    step.checkout(),
    step.action('shivammathur/setup-php@v2', {
      name: 'Setup PHP ${{ matrix.php-version }}',
      with: {
        'php-version': '${{ matrix.php-version }}',
        extensions: 'mbstring, intl, json, xml',
        tools: 'composer:v2',
        coverage: 'none',
      },
    }),
    step.run('echo "composer-cache-dir=$(composer config cache-files-dir)" >> $GITHUB_OUTPUT', {
      id: 'composer-cache',
      name: 'Get Composer cache directory',
    }),
    step.cache({
      path: '${{ steps.composer-cache.outputs.composer-cache-dir }}',
      key: `composer-\${{ runner.os }}-\${{ matrix.php-version }}-${hashFiles('**/composer.json')}`,
      restoreKeys: ['composer-${{ runner.os }}-${{ matrix.php-version }}-'],
    }),
    step.run('composer install --no-interaction --no-progress --prefer-dist', { name: 'Install Composer dependencies' }),
    step.run('composer analyse', { name: 'Static analysis (PHPStan)' }),
    step.run('composer test',    { name: 'Unit tests (PHPUnit)' }),
  ],
}),
```

## Run locally

```bash
cd examples/php-app

# Install the Rehearse toolchain
npm install -g @rehearse/runner @rehearse/cli
npm install                  # gets @rehearse/ci as a devDep for rh

# Compile + run
rh ci compile
runner run .github/workflows/ci.yml
```

Expected first run: the runner clones `shivammathur/setup-php@v2` into
`.runner/actions/shivammathur__setup-php__v2/` (one-time) and provisions
PHP 8.2 / 8.3 / 8.4 inside three parallel matrix cells. Subsequent runs
reuse the cached action clone.

## Why this is interesting for the runner

Most local CI tools (including `act`) require you to either pre-install
the PHP versions you want to test against on the host, or to provide a
Docker image that has them. **The runner does neither** — it executes
the same JS action GitHub Actions uses, which downloads the requested
PHP version itself. You write the workflow once; it works in CI, in
`runner run`, and in `runner watch`.

## Push to GitHub

The generated YAML has zero Rehearse runtime dependency. Push the repo
to GitHub and the workflow runs on standard `ubuntu-latest` GitHub-hosted
runners — three matrix cells in parallel, each on its own runner.
