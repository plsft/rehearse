# GitGate — Project Orientation

GitGate is a **local-first runner for GitHub Actions workflows**. It reads
`.github/workflows/*.yml` and executes them on a developer machine —
two backends (host subprocess, container via Docker), parallel matrix
via per-cell git worktree, real `actions/cache` semantics, watch mode,
pre-push hook, JS-action runtime, local + remote composite actions, and
local reusable workflows. Free, Apache 2.0, source on
[github.com/plsft/gitgate](https://github.com/plsft/gitgate).

## Repo layout

| Path | What | Status |
|---|---|---|
| `runner/` | `@gitgate/runner` — the CLI and the runner core | npm, Apache 2.0 |
| `cli/` | `@gitgate/cli` — companion CLI (`gg`) for TS-pipeline workflows | npm, Apache 2.0 |
| `ts-ci/` | `@gitgate/ci` — TypeScript SDK that compiles to GitHub Actions YAML | npm, Apache 2.0 |
| `git-engine/` | `@gitgate/git-core` — pure-TypeScript git engine (powers `actions/checkout` in the runner) | npm, Apache 2.0 |
| `site/` | Marketing site at [gitgate.com](https://gitgate.com) (deploys as `gitgate-web` Workers Static Assets) | private |
| `bench/` | Cross-OS benchmark harness (runner vs `act` vs GitHub-hosted) | private |
| `poc/` | Test fixtures (`poc/fixtures/*.yml`) and a hono clone (`poc/playground/hono`) used by the bench | private |
| `scripts/` | Release tooling (`release.mjs` bumps all four packages in lockstep, tags, pushes) | private |
| `.gitgate/pipelines/` | Our own CI pipelines authored in TypeScript (compiled to `.github/workflows/`) | private |
| `.github/workflows/` | Auto-generated by `gg ci compile` from `.gitgate/pipelines/`. Do not edit by hand | generated |
| `old/` | Pre-pivot artifacts archived for reference. Not built, not deployed, not on npm | archive |

## Current state (as of v0.3.2)

- 4 packages on npm at `0.3.2`, all Apache 2.0
- Runner: 96 tests across 10 suites, all green
- Git engine: 162 tests across 8 suites, all green
- Compat: 100% on our own CI, 96.15% on `vitejs/vite` ci.yml, 97.18% on `honojs/hono` ci.yml
- Bench (Linux GH-hosted, warm): 4.78× faster than `act` on `our-ci`,
  9.09× on `node-matrix`, 30.50× on `service-postgres` (where `act` times out)

## Working in this repo

- Build everything: `pnpm turbo build`
- Test the runner: `pnpm --filter @gitgate/runner test`
- Run the runner against a workflow: `node runner/dist/cli.js run <workflow>`
- Bench: `pnpm tsx bench/compare.ts --skip-cold`
- Release a new version: `pnpm release patch` (bumps all 4 package.json files, tags, pushes; CI publishes to npm)
- Deploy the marketing site: from `site/`, `npm run deploy` (vite build + wrangler deploy to `gitgate-web` Worker)

## Pivot history

The project was originally an agent-governance / git-hosting platform
(see `old/CLAUDE.md` for the original PRD). It pivoted in May 2026 to
a local-first GH Actions runner because the runner was the strongest
asset and had the clearest path to traction. The TypeScript git engine
and TypeScript CI authoring SDK survived the pivot and ship today.
Everything else from the pre-pivot product moved to `old/` and is not
deployed, built, or published. Do not import from `old/`.
