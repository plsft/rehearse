Open-source. No hosted product.

Rehearse is a small, focused open-source project. Four packages on npm, one source repo, no signup, no account, no telemetry. Just a CLI and the YAML you already have.

What Rehearse is

Rehearse started as a question: your CI loop is `git push → wait → fail → repeat`. What if it ran on every save instead? That answer turned into a runner that reads your .github/workflows/*.yml, executes it locally, and finishes 5–9× faster than act on standard workflows — 30× on services, where act doesn't actually complete. Same YAML, no new config.

Around the runner, three companion packages ship under the same Apache 2.0 umbrella: @rehearse/ci (TypeScript pipeline SDK), @rehearse/git-core (pure-TypeScript git engine), and @rehearse/cli (the rh companion CLI). All four are on npm, all four version in lockstep, all four are independent — install only what you need.

What Rehearse isn't

  • Not a hosted product. There is no rehearse.sh backend that runs your CI for you. The runner runs on your laptop or your own infra.
  • Not a SaaS. No accounts, no API keys, no monthly fee, no rate limits.
  • Not a fork or rewrite of act. Different architecture, different goals — the bench page documents the differences.
  • Not a wrapper around the git binary. @rehearse/git-core is a from-scratch TypeScript implementation of the git protocol; it has zero native dependencies.
  • Not telemetry-instrumented. The runner doesn't phone home. Compat audits, bench numbers, and provenance information stay on your machine.
