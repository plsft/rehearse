Release notes.

All four packages ship in lockstep at the same version. Tags live at github.com/plsft/rehearse/releases.

v0.4.0

2026-05-05 · positioning + breaking SDK
⚠ Breaking — @rehearse/ci
  • — Runner.ubicloud() removed from @rehearse/ci. Migrate to Runner.custom('ubicloud-standard-4') for explicit Ubicloud labels, or Runner.github('ubuntu-latest') for portable workflows.
  • — Cost estimator simplified — drops the Ubicloud↔GitHub tier comparison; now reports per-run + monthly GitHub-hosted cost only.
Runner
  • — Matrix display expansion: ${{ matrix.os }} in runs-on and step.name now displays its substituted value (e.g. host · ubuntu-latest instead of literal template).
  • — --remote output reads vm=Xms instead of sprite=Xms.
  • — All --help text + comments updated to use "VM" instead of internal infrastructure naming.
Marketing site
  • — New /vs page comparing Rehearse vs Blacksmith / Ubicloud / RunsOn / BuildJet / Namespace.
  • — Repositioned site to lead with the loop ("CI feedback in seconds, before you push"), not the runner.
  • — Three-target wedge on homepage: 1.5s local · 5s Pro · 95s GH Actions.
  • — New customers, changelog, bench, and SEO landing pages.
Pro (deployed independently)
  • — Daemon v0.0.7: matrix cells now serialize on the VM (--max-parallel 1) to eliminate the bun-cache install race that affected matrix workflows.
  • — API streaming endpoint: chunk-boundary fix — successful runs no longer mis-logged as failures in the audit table.
  • — Dashboard audit log: sprite.* action prefixes display as vm.*.

v0.3.17

2026-05-04 · ubicloud purge

Sweeping cleanup of Runner.ubicloud(...) usage across all 5 example dirs (node-app, dotnet-app, php-app, python-api, composite-action-demo) and the SDK defaults. Switched to Runner.github('ubuntu-latest') as the universal portable default.

Stripped Ubicloud-specific framing from package READMEs, marketing site, and docs. Bench harness no longer maps ubicloud-* labels to act.

v0.3.16

2026-05-04 · CLI scaffold fixes

Fixed three bugs found during end-to-end audit of the demo cue sheets:

  • — rh ci init: detect Bun via bun.lock (text format) in addition to bun.lockb (binary).
  • — rh ci init templates: drop bun.build() and node.build() from defaults — bun init -y doesn't scaffold a build script.
  • — Runner planner: shim-first composite expansion check — fixes dtolnay/rust-toolchain on workflows using matrix expressions in inputs.

v0.3.14, v0.3.15

2026-05-04 · publish race recovery

Procedural — three near-simultaneous tag pushes (0.3.14, 0.3.15, 0.3.16) caused a parallel-CI race where the npm latest tag landed on 0.3.14 instead of 0.3.16. Recovered by bumping to 0.3.17 (single tag = single CI run = single publish).

v0.3.13

2026-05-04 · Pro --remote streaming

Runner --remote now uses the streaming endpoint (line-by-line stdout/stderr as it happens, not buffered to the end). --remote auto-detects git origin URL + HEAD SHA + monorepo subdir from cwd. --env-file ships secrets to the VM as ${{ secrets.* }}.

v0.3.0 → v0.3.12

2026-04 → 2026-05 · 0.3.x series

The 0.3.x series stabilized the runner around the current capability set:

  • — setup-dotnet shim: real install via Microsoft's dotnet-install.sh, cached side-by-side on persistent rootfs.
  • — JS-action runtime: runs.using: node20 actions auto-cloned at the requested ref, full INPUT_* / GITHUB_OUTPUT contract.
  • — Composite actions: local ./.github/actions/* + remote org/repo[/sub]@ref, with auto-clone.
  • — Local reusable workflows: uses: ./.github/workflows/foo.yml at job level.
  • — Pro VM: 11 languages verified end-to-end (Node, Bun, Deno, Python, Go, Java, Ruby, Rust, Elixir, Erlang, .NET).
  • — Bench harness: head-to-head against act on standard fixtures, reproducible via gh workflow run bench.yml.

v0.1 — v0.2

2026-03 → 2026-05 · pre-pivot, archived

The project began as an agent-governance / git-hosting platform. Pivoted in May 2026 to a local-first GitHub Actions runner because the runner was the strongest asset and had the clearest path to traction. The TypeScript git engine (@rehearse/git-core) and the TypeScript CI authoring SDK (@rehearse/ci) survived the pivot and ship today as the foundation under the runner. Pre-pivot history is in git log for the curious.
