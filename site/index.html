// Features

      
A real runner. Not a demo.

      

        Parallel jobs. Matrix. Services. Caching. Watch mode. Pre-push hook.
        All wired in. 96 tests across 10 suites in the runner package alone.
      

    


    

      

        

          ⚡
          
HOST BACKEND

        

        
Skip Docker entirely.

        

          When your job doesn't need a container, GitGate runs steps directly
          on the host. Reuses the developer's node_modules,
          .bun-install, and toolchain caches. act can't do this.
        

      


      

        

          📦
          
CONTAINER BACKEND

        

        
Services, properly networked.

        

          Postgres, Redis, MySQL — start as services on a private Docker network with
          --network-alias wired correctly. Health-check
          waiting included. act stalls on this on Windows.
        

      


      

        

          ⊞
          
MATRIX

        

        
strategy.matrix works.

        

          Cartesian product, include,
          exclude. Each cell with substituted
          ${{ matrix.* }}. Cells run in
          parallel via per-cell
          git worktree — no shared-workspace races.
        

      


      

        

          ↻
          
WATCH MODE

        

        
Inner-loop dev tool.

        

          runner watch re-runs your workflow on
          every save. Debounced, queue-coalesced. Stop typing
          git push -f; iterate against your CI
          on every keystroke.
        

      


      

        

          🛡
          
PRE-PUSH HOOK

        

        
Block bad pushes at the source.

        

          runner install-hook writes a
          .git/hooks/pre-push that runs your
          workflow first. Push only if green. CI minutes spent: zero.
        

      


      

        

          🗃
          
CACHE

        

        
actions/cache works.

        

          Real actions/cache semantics on local
          fs. Exact-key hits. Restore-key longest-prefix matching. cache-hit,
          cache-primary-key,
          cache-matched-key outputs. Persistent across runs.
        

      


      

        

          ⊕
          
COMPOSITE ACTIONS

        

        
Local and remote, both expand.

        

          ./.github/actions/* and
          org/repo[/sub]@ref composites get inlined
          with ${{ inputs.x }} substituted from the
          parent's with:. Remote refs auto-clone at
          the requested ref. Real workflows just work.
        

      


      

        

          ⌂
          
SHIMS

        

        
15+ actions run in-process.

        

          checkout, setup-node,
          setup-bun, setup-go,
          cache, etc. — handled directly in
          TypeScript instead of pulling a Docker image to wrap them.
        

      


      

        

          𝓣𝓢
          
TS PIPELINES

        

        
YAML optional. TypeScript supported.

        

          Write workflows in TypeScript with @gitgate/ci.
          Compile to plain GitHub Actions YAML.
          Convert existing YAML to TS in one command.
        

      

    

  







  

    

      
// How it works

      
Three commands. Same YAML.

    


    

      

        

          01
          
Install

        

        
npm i -g @gitgate/runner

        
Single binary called runner. Apache 2.0. No account, no signup.

      

      

        

          02
          
Run

        

        
runner run .github/workflows/ci.yml
runner run .github/workflows/ci.yml -j test

        
Backend chosen per job: host (fast) or container (parity).

      

      

        

          03
          
Wire it in

        

        
runner watch # inner-loop
runner install-hook # pre-push

        
Inner loop or pre-push enforcement. Or both.

SUPPORTED TODAY

        
    
          
  • ✓ All run: steps (bash / pwsh / cmd)
    • 
          
  • ✓ Top ~15 actions in-process: checkout, setup-*, cache, artifacts
    • 
          
  • ✓ strategy.matrix with include/exclude
    • 
          
  • ✓ needs: with parallel scheduling
    • 
          
  • ✓ if: on jobs and steps (subset)
    • 
          
  • ✓ services: with health checks
    • 
          
  • ✓ Local and remote composite actions (./.github/actions/* + org/repo@ref)
    • 
          
  • ✓ Local reusable workflows (uses: ./.github/workflows/foo.yml at job level)
    • 
          
  • ✓ JavaScript actions (runs.using: node20) — auto-cloned at ref
    • 
          
  • ✓ ${{ matrix… }}, env, secrets, needs, steps, runner
    • 
        

      

      

        
NOT YET

        
    
          
  • ○ Remote reusable workflows (org/repo/.github/workflows/foo.yml@ref) — local form works
    • 
          
  • ○ Docker actions (runs.using: docker) — JS-action runtime ships, Docker doesn't yet
    • 
          
  • ○ OIDC / id-token: write
    • 
          
  • ○ concurrency: cancellation
    •
