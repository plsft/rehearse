// Features

      
A real runner. Not a demo.

      

        Parallel jobs. Matrix. Services. Caching. Watch mode. Pre-push hook.
        All wired in. 62 tests passing in the runner package alone.
      

    


    

      

        

          ⚡
          
HOST BACKEND

        

        
Skip Docker entirely.

        

          When your job doesn't need a container, GitGate runs steps directly
          on the host. Reuses the developer's node_modules,
          .bun-install, and toolchain caches. act can't do this.
        

      


      

        

          📦
          
CONTAINER BACKEND

        

        
Services, properly networked.

        

          Postgres, Redis, MySQL — start as services on a private Docker network with
          --network-alias wired correctly. Health-check
          waiting included. act stalls on this on Windows.
        

      


      

        

          ⊞
          
MATRIX

        

        
strategy.matrix works.

        

          Cartesian product, include,
          exclude. Each cell with substituted
          ${{ matrix.* }}. Independent jobs run
          in parallel; cells share workspace and run sequentially (per-cell
          worktree on the roadmap).
        

      


      

        

          ↻
          
WATCH MODE

        

        
Inner-loop dev tool.

        

          gitgate watch re-runs your workflow on
          every save. Debounced, queue-coalesced. Stop typing
          git push -f; iterate against your CI
          on every keystroke.
        

      


      

        

          🛡
          
PRE-PUSH HOOK

        

        
Block bad pushes at the source.

        

          gitgate install-hook writes a
          .git/hooks/pre-push that runs your
          workflow first. Push only if green. CI minutes spent: zero.
        

      


      

        

          🗃
          
CACHE

        

        
actions/cache works.

        

          Real actions/cache semantics on local
          fs. Exact-key hits. Restore-key longest-prefix matching. cache-hit,
          cache-primary-key,
          cache-matched-key outputs. Persistent across runs.
        

      


      

        

          ⊕
          
COMPOSITE ACTIONS

        

        
Local ./.github/actions/* expand.

        

          Composite actions get inlined with ${{ inputs.x }}
          substituted from the parent's with:. Real
          workflows that ship their own helpers just work.
        

      


      

        

          ⌂
          
SHIMS

        

        
15+ actions run in-process.

        

          checkout, setup-node,
          setup-bun, setup-go,
          cache, etc. — handled directly in
          TypeScript instead of pulling a Docker image to wrap them.
        

      


      

        

          𝓣𝓢
          
TS PIPELINES

        

        
YAML optional. TypeScript supported.

        

          Write workflows in TypeScript with @gitgate/ci.
          Compile to plain GitHub Actions YAML.
          Convert existing YAML to TS in one command.
        

      

    

  







  

    

      
// How it works

      
Three commands. Same YAML.

    


    

      

        

          01
          
Install

        

        
npm install -g gitgate

        
Single binary. Apache 2.0. No account, no signup.

      

      

        

          02
          
Run

        

        
gitgate run .github/workflows/ci.yml
gitgate run .github/workflows/ci.yml -j test

        
Backend chosen per job: host (fast) or container (parity).

      

      

        

          03
          
Wire it in

        

        
gitgate watch # inner-loop
gitgate install-hook # pre-push

        
Inner loop or pre-push enforcement. Or both.

      

    

  







  

    

      
// When you do need cloud

      
Optional cloud runners.
Bring your own Ubicloud.

      

        Most CI runs belong on the laptop. Sometimes you genuinely need cloud
        — bigger matrices, GPU runners, deploy steps. GitGate Cloud relays
        the run to Ubicloud
        — 10× cheaper than GitHub-hosted — using
        your own API key. We orchestrate; you pay
        Ubicloud direct.
      

      
    
        
  • ✓ Same runner binary, --cloud flag
    • 
        
  • ✓ Bring your own Ubicloud API key
    • 
        
  • ✓ Compute billed direct by Ubicloud (no GitGate margin on runner minutes)
    • 
        
  • ✓ Same cache and artifact contract as local
    • 
      

      

        Read the cloud docs →
