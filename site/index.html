// TypeScript-first CI

      
Author CI in TypeScript. Compile to YAML. Lossless.

      

        YAML for CI breaks down past a few hundred lines — no types, no
        autocomplete, copy-paste errors that surface 90 seconds into a
        run. @rehearse/ci
        gives you typed builders that compile to standard GitHub Actions YAML
        you can ship to stock GH Actions with zero runtime dependency on us.
      

    


    

      

        
// .rehearse/pipelines/ci.ts
import { pipeline, job, step, triggers, Runner } from '@rehearse/ci';
import { node } from '@rehearse/ci/presets';

export const ci = pipeline('CI', {
  triggers: [triggers.pullRequest(), triggers.push({ branches: ['main'] })],
  jobs: [
    job('test', {
      runner: Runner.github('ubuntu-latest'),
      matrix: { variables: { node: ['18', '20', '22'] } },
      steps: [
        step.checkout(),
        node.setup('${{ matrix.node }}'),
        node.install(),
        node.test(),
      ],
    }),
  ],
});

        
~17 lines of typed TS. Refactor across 100 pipelines without breaking one.

      

      

        

          
★ FORWARD PATH (TS → YAML)

          
100% lossless. Every public SDK feature compiles to canonical GitHub Actions YAML. Snapshot-tested against 6 reference workflows covering matrix, services, concurrency, defaults, environment, permissions, outputs, conditions, schedule, dispatch inputs.

        

        

          
★ MIGRATION ON-RAMP (YAML → TS)

          
One command: rh ci convert .github/workflows/ci.yml. Common shapes (triggers, jobs, runner, steps, env, conditions, outputs, permissions) survive the round-trip; advanced bits (matrix, services, concurrency, defaults) get hand-ported. Adopt typed CI on an existing repo without rewriting from scratch.

        

        

          
★ ZERO LOCK-IN

          
The compiled YAML has zero runtime dependency on @rehearse/ci. Delete the package, keep the YAML — your CI keeps running on stock GitHub Actions exactly as before.

// Features

      
What you get that hosted runners can't give you.

      

        These are the differentiators. Hosted-runner replacements
        (Blacksmith, Ubicloud, RunsOn) start the clock at git push —
        they can't compete on the inner-loop features no matter how
        cheap their per-minute price gets.
      

    


    

      
      

        

          ↻
          
★ WATCH MODE

        

        
CI on every save.

        

          rh watch re-runs your workflow on
          every file change. Debounced, queue-coalesced. No hosted runner can do this —
          they bill per minute and you'd go bankrupt.
        

      


      

        

          🛡
          
★ PRE-PUSH HOOK

        

        
Block bad pushes at the source.

        

          rh install-hook writes a
          .git/hooks/pre-push that runs your
          workflow first. Push only if green. CI minutes burned: zero.
        

      


      

        

          ⏎
          
★ NO LOCK-IN

        

        
OSS-first, ejectable.

        

          Apache 2.0, no signup, no telemetry, no phone-home. The
          compiled YAML works on stock GitHub Actions with no Rehearse
          runtime. If we vanish, your CI doesn't notice.
        

      


      
      

        

          ⚡
          
HOST BACKEND

        

        
Skip Docker entirely.

        

          When your job doesn't need a container, Rehearse runs steps directly
          on the host. Reuses the developer's node_modules,
          .bun-install, and toolchain caches. act can't do this.
        

      


      

        

          📦
          
CONTAINER BACKEND

        

        
Services, properly networked.

        

          Postgres, Redis, MySQL — start as services on a private Docker network with
          --network-alias wired correctly. Health-check
          waiting included. act times out on this.
        

      


      

        

          ⊞
          
MATRIX

        

        
strategy.matrix works.

        

          Cartesian product, include,
          exclude. Each cell with substituted
          ${{ matrix.* }}. Cells run in
          parallel via per-cell
          git worktree — no shared-workspace races.
        

      


      

        

          🗃
          
CACHE

        

        
actions/cache works.

        

          Real actions/cache semantics on local
          fs. Exact-key hits. Restore-key longest-prefix matching. cache-hit,
          cache-primary-key,
          cache-matched-key outputs. Persistent across runs.
        

      


      

        

          ⊕
          
COMPOSITE ACTIONS

        

        
Local and remote, both expand.

        

          ./.github/actions/* and
          org/repo[/sub]@ref composites get inlined
          with ${{ inputs.x }} substituted from the
          parent's with:. Remote refs auto-clone at
          the requested ref. Real workflows just work.
        

      


      

        

          ⌂
          
SHIMS

        

        
15+ actions run in-process.

        

          checkout, setup-node,
          setup-bun, setup-go,
          cache, etc. — handled directly in
          TypeScript instead of pulling a Docker image to wrap them.
        

      


      

        

          𝓣𝓢
          
TS PIPELINES

        

        
YAML optional. TypeScript supported.

        

          Write workflows in TypeScript with @rehearse/ci.
          Compile to plain GitHub Actions YAML.
          Convert existing YAML to TS in one command.
