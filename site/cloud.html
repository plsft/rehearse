// Setup

From local to cloud in 60 seconds.

Three commands. No GitHub App, no webhook configuration.

01
Get a Ubicloud API key

Sign up at ubicloud.com, create a project, mint an API key under Settings → Tokens. 

export UBICLOUD_TOKEN=ubi_xxxxxxxxxxxxxxxxxxxx
export UBICLOUD_PROJECT=your-project
02
Add the cloud plugin

Authenticate once. The plugin registers your Ubicloud project with GitGate and issues a relay token. 

gitgate cloud login
# Detects UBICLOUD_TOKEN, validates, registers your project

gitgate cloud status
✓ Authenticated as you@your.org
✓ Ubicloud project: your-project (us-east-1)
✓ Relay: gitgate-cloud-us-east-1.gitgate.com
03
Run a workflow on cloud

Same command, add --cloud. The runner uploads the workflow, schedules a Ubicloud VM, streams logs back. 

gitgate run .github/workflows/release.yml --cloud
# streams cloud-side stdout to your terminal

gitgate run … --cloud --runner ubicloud-standard-8
# pick a specific runner size

gitgate run … --cloud --max-parallel 8
# run all matrix cells in parallel on cloud (paid)
// When to use cloud

A clear decision tree.

USE LOCAL
…for the pre-push loop

Anything you'd run as a sanity check before pushing. Lint, typecheck, unit tests, build, smoke test. 90% of CI runs are this. Local is faster, free, and offline-capable.

USE CLOUD
…for parallel matrices and deploys

A 12-cell matrix that takes 30 minutes locally takes 3 minutes across 12 cloud runners. Production deploy steps that shouldn't run on a laptop. Anything that needs an isolated network.

// Architecture

No magic. Just orchestration.

The CLI uploads a workflow bundle to a regional GitGate relay. The relay uses your Ubicloud API key (server-side, never logged) to provision a runner, runs the workflow with the same backend logic as local, and streams logs back. Artifacts and cache live in your Cloudflare R2 bucket (or ours by default).

data flow 
your-laptop                gitgate-relay              your-ubicloud-vm
─────────────              ─────────────              ────────────────
gitgate run --cloud
   └─ workflow bundle ──>  validate, sign
                           use UBICLOUD_TOKEN ────>   provision VM
                           stream stdout      <────  run steps
                           upload cache+arts
your-laptop  <───────────  stream stdout ───────────  done
                           tear down VM      ────>   release
