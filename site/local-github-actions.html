// Why run GitHub Actions locally

The CI loop you actually want.

  • 1
    Catch failures before git push. GH Actions runs after push, with 30-90s spin-up tax. Run the same workflow locally and the loop closes in seconds — fix bugs before they hit your branch protection.
  • 2
    Iterate at the speed of your tests. rh watch reruns CI on every save. Edit a test, hit save, see it green or red without leaving the editor. Same YAML you commit; no separate dev script.
  • 3
    Burn zero CI minutes. The runner uses your laptop's CPU. No cloud bill. No wasted runner-pool minutes. Pre-push hook (rh install-hook) gates pushes on a green local run, so what hits GitHub is already known to pass.
  • 4
    Test cross-OS and matrix without 3× runner spin-up. strategy.matrix works exactly like on GitHub — cells run in parallel via per-cell git worktree, no shared-workspace races. 9-cell matrix runs on your laptop in a fraction of the time GH Actions takes.
  • 5
    No edits to your YAML. The runner reads .github/workflows/*.yml directly. 97% of real workflows just work — we audited vitejs/vite (96.2%), honojs/hono (97.2%), and run our own CI at 100%.
// Example · 30-second walkthrough

From git clone to green CI in 30 seconds.

Below is a real Bun-based GitHub Actions workflow with prettier + eslint + tsc + test + build, run locally:

1. Your existing workflow YAML

# .github/workflows/ci.yml — works on GitHub-hosted AND locally
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest
      - run: bun install --frozen-lockfile
      - run: bunx prettier --check .
      - run: bunx eslint .
      - run: bunx tsc --noEmit
      - run: bun test
      - run: bun build src/index.ts --outdir dist --target bun

2. Run it locally

npm install -g @rehearse/cli
cd ~/your-repo
time rh run .github/workflows/ci.yml

// Going further

Watch mode + pre-push hook.

// rh watch

CI on every save.

rh watch .github/workflows/ci.yml

Watches your repo and re-runs the workflow on every file save. Debounced and queue-coalesced — rapid edits collapse into a single rerun. Your IDE save becomes your CI trigger. No competitor in the hosted-runner SaaS layer can match this — they bill per minute and would bankrupt you.

// rh install-hook

Block bad pushes at the source.

rh install-hook

Writes a .git/hooks/pre-push that runs your CI workflow before letting git push complete. Push only if green. The CI minutes you save are real — broken pushes never reach GitHub.
