Pro quickstart.

Pass --remote to the OSS runner with your Pro token set. Workflow ships to your dedicated sprite; cache survives between runs.

01 — sign up

Buy Pro at /pricing. Stripe sends you a receipt; we send you the API key (shown once on the success page and in the welcome email). Save it somewhere safe.

Right after checkout we provision your dedicated microVM. Takes 30-60 seconds. The welcome email goes out only when your sprite is up and ready to accept runs.

02 — set the token

export REHEARSE_TOKEN=rh_pro_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

03 — run remote

runner run --remote .github/workflows/ci.yml

The runner POSTs your workflow YAML to api.rehearse.sh/v1/runs/stream, which forwards to your sprite. The sprite executes the same YAML you'd run locally and streams logs back line-by-line as the run progresses.

Drop the --remote flag and you're back to local execution. Same workflow file, same output format. No lock-in.

04 — caches accumulate

Your sprite's /home/sprite/.npm persists across runs. After the first run, subsequent runs that touch the same packages skip the network. We measured a typical install dropping 10.5s → 5.1s (~2×).

Other caches that persist for free: ~/.cache/pip, ~/.cargo/registry, ~/.gradle, ~/.bun/install/cache, your node_modules, the cloned repo. Anything you write to disk under /home/sprite/ survives.

05 — manage from the dashboard

At pro.rehearse.sh/dashboard: rotate API keys, see recent runs, hit Stripe Customer Portal to manage billing.

troubleshooting

  • — "runner not provisioned": your sprite is still spinning up after signup. Provisioning takes 60–120s after Stripe checkout. Wait, then retry. If it persists, email pro@rehearse.sh.
  • — "401 Unauthorized": token revoked or wrong. Check echo $REHEARSE_TOKEN; create a new key in the dashboard.
  • — slow first run: expected — your sprite has no cache yet. Run 2 onward is ~2× faster on install steps.
  • — need to clear cache: currently a manual ask — email pro@rehearse.sh and we'll wipe and re-provision. Self-serve reset lands in v1.
