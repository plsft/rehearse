Pro quickstart.

Pass --remote to the OSS runner with your Pro token set. Workflow ships to your dedicated sprite; cache survives between runs.

01 — sign up

Buy Pro at /pricing. Stripe sends you a receipt; we send you the API key (shown once on the success page and in the welcome email). Save it somewhere safe.

Right after checkout we provision your dedicated microVM. Takes 30-60 seconds. The welcome email goes out only when your sprite is up and ready to accept runs.

02 — set the token

export REHEARSE_TOKEN=rh_pro_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

03 — run remote

rehearse run --remote .github/workflows/ci.yml

Runner POSTs your workflow YAML to api.rehearse.sh/v1/runs, which forwards it to your sprite. Sprite executes the same YAML you'd run locally, returns logs.

Drop the --remote flag and you're back to local execution. No other code changes.

04 — see your cache fill up

After the first run, your sprite has the project's ~/.npm populated. Run 2 hits cache; npm ci typically goes from 30-90s to 3-10s.

The dashboard at pro.rehearse.sh/dashboard/usage shows cache size, run history, and bandwidth.

05 — rotate keys

In Dashboard → API keys: create a new key, swap your env-var, revoke the old one. Revocation is immediate; cached tokens expire within 5 minutes.

troubleshooting

  • — "runner not provisioned": sprite is still spinning up after signup, or onboarding paused. Wait 60s and retry; if it persists email pro@rehearse.sh.
  • — "401 Unauthorized": token revoked or wrong. echo $REHEARSE_TOKEN; create a new key in the dashboard.
  • — slow first run: expected — your sprite has no cache yet. Run 2 onward should be ~5-10× faster on the install step.
  • — need to clear cache: from the dashboard, "Reset cache" wipes ~/.npm etc. Sometimes useful when you've changed lockfile structure.
