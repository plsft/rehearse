Pro quickstart.

Set the token, the OSS runner does the rest. No new commands.

01 — sign up

Buy Pro at /pricing. Stripe sends you a receipt; we send you the API key (shown once on the success page and in the welcome email). Save it somewhere safe.

02 — set the token

Either via env-var or a config file. Both work; env-var wins if both are present.

export REHEARSE_TOKEN=rh_pro_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Or in rehearse.config.mjs:

export default {
  images: {
    source: 'rehearse-pro',
    apiKey: process.env.REHEARSE_TOKEN,
    fallback: 'public',
  },
};

03 — run as before

rehearse run .github/workflows/ci.yml

The runner pulls warmed images automatically when your workflow uses Node, Python, Go, etc. If anything goes wrong (token invalid, image not in catalog, registry unreachable) the runner falls back to public Docker Hub silently — your CI never fails because of Pro.

04 — verify a Pro image (optional)

Every image is cosign-signed via Sigstore. To verify before pulling:

cosign verify registry.rehearse.sh/node:20-warm \
  --certificate-identity-regexp 'https://github.com/rehearse-sh/images/' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com'

05 — rotate keys

In Dashboard → API keys: create a new key, swap your env-var, revoke the old one. Revocation is immediate; cached tokens expire within 5 minutes.

troubleshooting

  • — "401 Unauthorized" on pull: token revoked or wrong. Run echo $REHEARSE_TOKEN; create a new key in the dashboard.
  • — "image not in catalog": the requested name:tag isn't a Pro image. Check /catalog; the runner falls back to public Docker Hub silently.
  • — slow pulls: first pull is uncached at the edge — subsequent pulls hit the Worker's cache. If you're seeing >5s on small manifests after the first request, email us.
