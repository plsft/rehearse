Run your CI locally in 30 seconds.

Install, point at a workflow, watch it run. Below: install, every-day commands, and the topics that come up once you've used it for a week.

Quickstart

INSTALL
# Requires Node 22+ — install via Homebrew if you don't have it:
brew install node
npm install -g @rehearse/cli
rh --version
0.6.5
# Requires Node 22+. Install via your distro's package manager:
sudo apt install nodejs npm     # Debian/Ubuntu
sudo pacman -S nodejs npm        # Arch
sudo dnf install nodejs npm      # Fedora
npm install -g @rehearse/cli
rh --version
0.6.5
# Requires Node 22+ (PowerShell or Git Bash):
winget install OpenJS.NodeJS    # or use nvm-windows / scoop
npm install -g @rehearse/cli
rh --version
0.6.5

Windows tip: launch rh from the same shell you'd use to run your CI scripts. Bash steps inside run: blocks need Git Bash on PATH; shell: pwsh works natively in PowerShell.

RUN A WORKFLOW
rh run .github/workflows/ci.yml                       # all jobs
rh run .github/workflows/ci.yml --job test            # one job
rh run .github/workflows/ci.yml --matrix os=ubuntu-latest  # filter cells
rh run … --backend container                          # force docker
rh run … --max-parallel 8                             # concurrency knob
rh run … --verbose                                    # stream all step output
rh run … --quiet                                      # minimal, scripting
INTEGRATE
rh watch .github/workflows/ci.yml             # inner-loop, re-runs on save
rh install-hook                               # pre-push git hook
rh compat .github/workflows/ci.yml            # static audit (% would execute)
PRO (--remote)
export REHEARSE_TOKEN=rh_pro_live_…           # bash/zsh
$env:REHEARSE_TOKEN = "rh_pro_live_…"          # PowerShell
rh run --remote .github/workflows/ci.yml      # ship to your VM
rh run --remote … --matrix os=ubuntu-latest   # forwarded to VM
TYPESCRIPT PIPELINES
rh ci init                                   # scaffold .rehearse/pipelines/
rh ci compile                                # .ts → .github/workflows/*.yml
rh ci convert .github/workflows/ci.yml       # existing YAML → TS (migration)
rh ci validate                               # type-check pipelines
rh ci watch                                  # recompile on save
rh ci estimate                               # GH-hosted CI cost (per-run + monthly)

// Core

Run a workflow

rh run <path> reads a GitHub Actions YAML, plans the jobs, picks a backend per job, and executes. The default backend is auto: host for plain run: work, container for jobs that declare services: or specify a container:.

flags 
-j, --job <name>          run only this job (matrix variants of it still all run)
-b, --backend <type>      host | container | auto (default: auto)
-p, --max-parallel <n>   max concurrent jobs (omit to let the scheduler pick — typically min(cpus, 4))
-c, --cwd <dir>          working directory (default: inferred from workflow path)
    --fail-fast           cancel sibling jobs on first failure
    --quiet               minimal output (machine-readable result only)
    --bench               single JSON line on stdout (for benchmarks)
    --env-file <file>     load env vars from KEY=VALUE file
// Core

Watch mode

rh watch <path> re-runs the workflow on every change to .yml, .ts, .js, or .json files in the repo. Debounced; if a re-run is in flight when a change lands, the next run is queued and coalesced.

// Core

Pre-push hook

rh install-hook writes a .git/hooks/pre-push that runs your workflow first. Push only if green. CI minutes spent: zero. 

rh install-hook
rh install-hook -w .github/workflows/ci.yml -j test
cat .git/hooks/pre-push
# #!/usr/bin/env bash
# # Auto-generated by 'rh install-hook'.
# exec npx -p @rehearse/cli rh run ".github/workflows/ci.yml" --fail-fast
// Features

Matrix

Full strategy.matrix support: cartesian product of variables, plus include / exclude. ${{ matrix.* }} substitutes per-cell into run, env, with, and if.

✓ Matrix cells run in parallel: each cell gets its own git worktree off the host repo (with node_modules and other toolchain caches symlinked through). No shared-workspace races on coverage/, build artifacts, etc.

// Features

Services

Jobs that declare services: auto-select the container backend. Each service starts on a private Docker network with --network-alias <name> — your steps reach it as postgres or redis. Health-check options are forwarded to docker run; the runner waits for healthy before the first step.

// Features

Caching

actions/cache, .../cache/restore, and .../cache/save all work. Storage is local fs under .runner/cache/, keyed by SHA-256 of the cache key, with restore-key longest-prefix matching. Outputs match GitHub: cache-hit, cache-primary-key, cache-matched-key.

// SDK

TypeScript pipelines

Optional. Author your workflow in TypeScript with @rehearse/ci, compile to standard GitHub Actions YAML. Type-safe steps, IDE autocomplete, real refactor support. The compiled YAML has zero Rehearse dependency at CI time. 

import { pipeline, job, step, triggers, Runner } from '@rehearse/ci';
import { node } from '@rehearse/ci/presets';

export const ci = pipeline('CI', {
  triggers: [triggers.pullRequest()],
  jobs: [
    job('test', {
      runner: Runner.github('ubuntu-latest'),
      steps: [step.checkout(), node.setup('20'), node.install(), node.test()],
    }),
  ],
});

Convert existing YAML to TypeScript with rh ci convert .github/workflows/ci.yml. Compile back to YAML with rh ci compile.
