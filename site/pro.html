Real numbers on the same workload.

A workflow that runs npm install typescript vite vitest react react-dom express axios zod hono — eight popular packages, no project deps. Three scenarios, all measured this morning.

scenario install wall notes
Local Docker, cold pull + install — 22.0s first docker run on a clean machine
Local Docker, image cached 17.1s 17.1s image cached, no npm cache
Pro cold cache 10.5s 12.5s first run on a fresh sprite
Pro warm cache 5.1s 6.7s second + subsequent runs

Pro warm vs local Docker cached: 2.55× faster. Cold-to-warm Pro speedup: 2.07× on install, 1.86× on wall. The real savings show up when you run CI dozens of times a day, which is the whole point of CI.

Reproduce: runner run --remote bench/pro-bench.yml against your Pro account, twice. The first run is cold; the second is warm. Exact ms vary by region and upstream-registry latency — the cold/warm ratio is what's load-bearing.

// quickstart

Three commands.

# 1. install the OSS runner
npm install -g @rehearse/runner@latest

# 2. set your token (after Stripe checkout)
export REHEARSE_TOKEN=rh_pro_live_…

# 3. run any workflow that targets a supported language (see below)
runner run --remote .github/workflows/ci.yml

Same command without --remote runs locally. Drop the flag any time — there's no lock-in.

// honest caveats

What Pro is not.

  • — 11 languages today, PHP not yet. Pro v0 verified end-to-end against your Pro account on Node, Bun, Deno, Python, Go, Java, Ruby, Rust, Elixir, Erlang, and .NET. PHP is the only language not yet supported (Sprites.dev doesn't ship PHP and there's no clean install path); those workflows still run great on the OSS runner locally.
  • — No service containers (yet). Workflows that declare services: (Postgres, Redis, etc.) need Docker on the sprite, which isn't available on the default tier. Use the OSS runner with the container backend locally for those.
  • — First run isn't faster. The cache populates on run 1; runs 2-N benefit. If your project sees its first CI run on Pro and never again, you'll see no win.
  • — Not a parallel-matrix multiplier. One sprite per team in v0. Matrix jobs run sequentially on your sprite. Team plans add parallelism in v1.
  • — Caches are private to your sprite. No cross-team sharing — that's the security boundary. If you have 5 unrelated repos, all 5 share one cache, but no other customer touches yours.
  • — Not free. $49/mo. The OSS runner is and always will be Apache 2.0 and free.
