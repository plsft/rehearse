// Honest comparison

Rehearse vs hosted-runner replacements.

Comparison-shopping CI runners is annoying. We did it for you. Honest take, not vendor FUD — including where Blacksmith / Ubicloud / RunsOn are the right answer for you.

// TL;DR

Different shape entirely.

Blacksmith, Ubicloud, RunsOn, BuildJet, Namespace — these are hosted-runner replacements. They start the clock at git push and give you a faster, cheaper GH-hosted runner pool. They compete with each other on price-per-minute, instance specs, and cache features.

Rehearse is in a different category. The OSS runner gives you sub-second CI feedback on a laptop, on every save, before you push. The Pro VM is what you offload to when local isn't enough — $49/mo for 25,000 active CPU minutes (+$49 per additional 25k block), single-tenant, whole-rootfs persistence.

If you're shopping for a "faster GitHub Actions runner," Blacksmith and Ubicloud are good options — drop-in GH replacements with per-minute pricing meaningfully cheaper than GitHub-hosted. Use one of them.

If you're shopping for "CI feedback that's faster than `git push → wait`," you have two real options: nektos/act (the OSS pioneer in this space, MIT, ~56k stars) and Rehearse. Both run your .github/workflows/*.yml locally. We compete with act on speed (5–30× faster — see /bench), feature coverage (host backend / services / watch mode / pre-push hook), and an optional hosted-offload target via the same binary. Pick the one that fits your workflow.

What's true about hosted-runner replacements (Blacksmith / Ubicloud / RunsOn / Depot / WarpBuild): none of them try to do local execution at all. They bill per minute and "rehearse watch on every save" would bankrupt you. So the inner loop is the OSS-runner game (act and Rehearse), not theirs.

// Honest take

When to pick the other guys.

The hosted-runner SaaS category has real winners — we'd rather be honest about when they're the right call than have you discover it three months in.

PICK BLACKSMITH IF:
  • — You want a drop-in GH-hosted replacement, deployed in <5 min by changing a runs-on: label
  • — Your CI is fully post-push and you have no laptop-CI workflow at all
  • — You need bare-metal performance for compile-bound builds (Rust, Swift)
  • — Your usage stays under their free-tier ceiling (a few thousand minutes/month)
PICK UBICLOUD IF:
  • — You want OSS on the runner side too (Ubicloud has open-source components)
  • — You need ARM runners at scale (their pricing is good there)
  • — You want self-hostable runner infrastructure as a fallback option
  • — You have an existing Ubicloud GitHub App in your org and want to keep things simple
PICK REHEARSE IF:
  • — You want CI feedback before git push, not after
  • — Your inner-loop iteration speed is your bottleneck
  • — You hate "push, wait 5 min, see one typo, fix, push, wait 5 min"
  • — You author CI in TypeScript (or want to)
  • — You want OSS so vendor risk is zero
  • — You're heavy enough that flat-rate < per-minute
  • — You want one tool that covers local + offload + GH

Best combo for many teams: use the Rehearse OSS runner locally for the inner loop (free), plus Blacksmith / Ubicloud as your post-push GH-hosted replacement. They solve different problems. Use both if it fits your shape.
