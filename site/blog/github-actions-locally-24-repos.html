Running GitHub Actions locally on 24 repos: notes from one week

TL;DR

I'm dogfooding Rehearse on 24 of my own production repos this week — internal tools, OSS side projects, and the marketing site itself. This is the public running tally: what worked first try, what broke, what surprised me, what I'd change. Posted in real time so it's honest. Updated as I go.

Why dogfood?

Rehearse v0.5.0 just shipped. I have one user (me). To find the bugs that don't show up in synthetic tests, I need real workflows from real repos doing real work. I picked 24 of my own repos that span: Node + Bun + Python + Go + .NET + Rust; solo and team projects; monorepo and single-package; trivial CI and complex matrix CI; workflows with services and without.

The methodology

For each repo:

  1. cd ~/projects/repo-name
  2. rehearse compat .github/workflows/ci.yml — static audit, what % executes
  3. time rehearse run .github/workflows/ci.yml — local execution, time it
  4. time rehearse run --remote .github/workflows/ci.yml — Pro VM, time it
  5. Compare to the latest GitHub Actions run from the same SHA
  6. Note any failures, surprises, broken expectations

What I'm watching for

  • Workflows that crash — these are real bugs, not edge cases. Each gets a GitHub issue.
  • Workflows that work but slowly — inefficient code paths in the runner, fixable.
  • "Huh, that's annoying" — UX rough edges. Less critical than bugs but accumulating these is what makes a tool feel polished.
  • "I wish it did X" — feature gaps. Often resolved by reading existing docs more carefully.
  • "This actually saved me time" — the bar I'm trying to clear. If 80% of repos hit this bar, the product is real.

Why this matters more than the bench

The bench page is reproducible — anyone can run it. But synthetic benchmarks miss real-world workflow shapes: the workflow that uses an obscure JS action, the workflow with weird secret-passing patterns, the monorepo workflow with 10 jobs and complex needs: dependencies.

Dogfooding 24 production repos exercises the long tail. Whatever I find here is what a stranger would find in week 1.
