// security

Trust surface: image bytes.

The single sentence we want every CISO to hear: Rehearse Pro never holds customer source code, secrets, build artifacts, or run logs. We serve container image bytes. That's the entire surface.

// what we store
  • — Email + Stripe customer ID + Stripe subscription metadata
  • — Hashed API keys (sha256, prefix indexed for lookup)
  • — Per-pull events: team id, key id, image id, byte count, timestamp
  • — Bandwidth usage aggregates per team per month
  • — Audit log entries for privileged actions (key create, key revoke, billing changes)
// what we never store
  • — Source code
  • — Build artifacts or run logs
  • — CI environment variables or secrets
  • — API key plaintext (shown once at creation, then sha256 only)
  • — IP addresses (we hash to a 16-char prefix for abuse detection)
// image integrity
  • — Cosign-signed via Sigstore keyless OIDC. Verify with cosign verify registry.rehearse.sh/<image>.
  • — Trivy-scanned on every build. Fixed CRITICAL CVEs gate the build — no image with a fixable CRITICAL ever ships. Fixed HIGH CVEs are advisory and reviewed weekly; we don't gate HIGH because Debian's patch cadence makes that catalog-breaking. We tighten as our patch ops catch up.
  • — Multi-arch reproducible: amd64 + arm64 from the same Dockerfile.
  • — Weekly rebuild: every Monday 06:00 UTC. Out-of-band rebuilds for critical CVE disclosures.
  • — Public Dockerfiles: every image's source lives at github.com/plsft/rehearse-images.
// infrastructure
  • — Cloudflare Workers + D1 + R2. Single-region for v1.
  • — Stripe handles all card data. We never see PANs.
  • — TLS terminated at Cloudflare. HSTS preload eligible.
  • — No customer data crosses to third parties beyond Stripe (billing) and Resend (transactional email).
// reporting a finding

Email security@rehearse.sh. We acknowledge within 24 hours and update you weekly until resolved. We don't yet run a paid bounty program; we will credit reporters in the changelog if requested.

SOC 2 Type 1 in flight; planning Type 2 12 months post-launch. DPA template available on request for Enterprise customers.
