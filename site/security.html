// security

A microVM only you ever touch.

Rehearse Pro provisions a hardware-isolated microVM for your team. Your code lives there, your caches accumulate there, and nobody else's runs ever touch your filesystem. We hold the orchestration layer and a 50KB metadata record per team — nothing more.

// where your code lives
  • — Inside your VM. Each team gets a dedicated microVM with hardware-level isolation. 100 GB persistent ext4 filesystem.
  • — No cross-tenant access. VMs have no shared kernel, no shared filesystem, no shared network namespace with other customers' VMs.
  • — Code transfers at run time. When you run rehearse run --remote, your workflow YAML is shipped to your VM over HTTPS. It executes there. Logs come back over HTTPS as they happen. Persistent files (your caches, repo clone) stay on the VM.
  • — Cancel = wiped. Cancel your subscription and we destroy your VM. The 100 GB volume is reclaimed.
// what we store on our control plane
  • — Email + Stripe customer/subscription metadata
  • — Hashed API keys (sha256, prefix indexed)
  • — Per-run records: team id, key id, exit code, duration, run timestamp
  • — Last 50 KB of run output (so you can view recent run logs from the dashboard)
  • — Audit log entries for privileged actions (key create, key revoke, billing changes)
// what we never store
  • — API key plaintext (shown once at creation, then sha256 only)
  • — IP addresses (we hash to a 16-char prefix for abuse detection)
  • — Source code from your repos (lives on your VM, not us)
  • — Long-tail run logs older than 30 days
// infrastructure
  • — Cloudflare Workers (api.rehearse.sh) + D1 (control plane database)
  • — Hardware-isolated microVM execution layer (vendor undisclosed for security; named in DPA on request)
  • — Stripe handles all card data. We never see PANs.
  • — TLS terminated at Cloudflare. HSTS preload eligible.
  • — No customer data crosses to third parties beyond Stripe (billing), our microVM compute provider, and Resend (email).
// reporting a finding

Email security@rehearse.sh. We acknowledge within 24 hours and update you weekly until resolved. We don't yet run a paid bounty program; we will credit reporters in the changelog if requested.

No SOC 2 today. We're a small early-stage shop and SOC 2 is a 12-month + audit-fee commitment we haven't earned the right to make yet. If you're a regulated buyer who needs SOC 2 to evaluate, talk to us — we'll discuss timeline + cost-share for accelerating it. DPA template available on request for any customer.
